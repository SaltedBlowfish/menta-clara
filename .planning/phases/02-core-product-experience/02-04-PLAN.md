1. Create `src/calendar/use-calendar.ts`:
   - Export `useCalendar(selectedDate: Date, onSelectDay: (date: Date) => void)` returning:
     ```typescript
     {
       daysWithNotes: ReadonlySet<string>; // Set of 'YYYY-MM-DD' strings
       displayedMonth: Date; // First day of displayed month
       goToNextMonth: () => void;
       goToPreviousMonth: () => void;
       monthLabel: string; // "March 2026"
     }
     ```
   - State: `displayedMonth` initialized to first day of `selectedDate`'s month using `startOfMonth` from date-fns
   - `goToNextMonth`: set displayedMonth to `addMonths(displayedMonth, 1)` from date-fns
   - `goToPreviousMonth`: set displayedMonth to `subMonths(displayedMonth, 1)` from date-fns
   - `monthLabel`: `format(displayedMonth, 'MMMM yyyy')` from date-fns
   - `daysWithNotes`: useEffect that queries IndexedDB when displayedMonth changes:
     - Get year and month from displayedMonth
     - Use `IDBKeyRange.bound('daily:YYYY-MM-01', 'daily:YYYY-MM-31')` to query note keys
     - Map keys to date strings (strip 'daily:' prefix)
     - Store in state as a Set for O(1) lookup
   - When `selectedDate` changes, update `displayedMonth` to match its month (so calendar follows navigation)

1. Create `src/shared/use-keyboard-shortcuts.ts`:
   - Export `useKeyboardShortcuts(shortcuts: ReadonlyArray<KeyboardShortcut>): void`
   - Type: `interface KeyboardShortcut { handler: () => void; key: string; meta: boolean }`
   - Registers a single `keydown` listener on `window` in a useEffect
   - For each shortcut, checks `event.metaKey === shortcut.meta && event.key === shortcut.key`
   - Calls `event.preventDefault()` before handler to prevent browser defaults (CMD+[ is browser back)
   - Cleanup: removes listener on unmount
   - Predefined shortcuts to register from app.tsx:
     - `{ key: '[', meta: true, handler: focusLeftPane }` -- CMD+[ focuses daily note editor (first focusable in `#daily-pane`)
     - `{ key: ']', meta: true, handler: focusRightPane }` -- CMD+] focuses first focusable in `#right-pane`
   - `focusLeftPane`: `document.querySelector('#daily-pane .tiptap')?.focus()` (the TipTap editor is contenteditable)
   - `focusRightPane`: find first expanded collapsible section's content, focus first focusable element within it. Use `document.querySelector('#right-pane button, #right-pane [contenteditable]')?.focus()`

2. Rewrite `src/app/app.tsx` to wire everything together:
   - Imports: SplitPane, DailyPane, WeeklySection, PermanentSection, CalendarSection, ThemeToggle, LiveRegion, useKeyboardShortcuts
   - State: `selectedDate: Date` initialized to `new Date()`, `announcement: string` initialized to `''`
   - `handleSelectDay` callback: `(date: Date) => { setSelectedDate(date); setAnnouncement('Viewing ' + format(date, 'MMMM d, yyyy')); }`
   - Register keyboard shortcuts via useKeyboardShortcuts:
     ```typescript
     useKeyboardShortcuts([
       { handler: () => { document.querySelector<HTMLElement>('#daily-pane .tiptap')?.focus(); }, key: '[', meta: true },
       { handler: () => { document.querySelector<HTMLElement>('#right-pane button, #right-pane [contenteditable="true"]')?.focus(); }, key: ']', meta: true },
     ]);
     ```
   - Render:
     ```tsx
     <>
       <ThemeToggle />
       <SplitPane
         left={<DailyPane date={selectedDate} />}
         right={
           <>
             <WeeklySection date={selectedDate} />

           </>
         }
       />
       <LiveRegion message={announcement} />
     </>
     ```
