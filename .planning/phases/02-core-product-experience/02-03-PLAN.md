1. Create `src/permanent/use-permanent-notes.ts`:
   - Export `usePermanentNotes()` returning:
     ```typescript
     {
       createNote: (name: string) => Promise<string>;
       deleteNote: (noteId: string) => Promise<void>;
       loading: boolean;
       notes: ReadonlyArray<{ id: string; name: string }>;
       selectedNoteId: string | null;
       selectNote: (noteId: string) => void;
     }
     ```
   - On mount: query IndexedDB for all keys with `permanent:` prefix using `IDBKeyRange.bound('permanent:', 'permanent:\uffff')`
   - Use `getDatabase().getAllKeys(NOTES_STORE, range)` to get note IDs
   - For note names: store a separate metadata entry in IndexedDB with key `meta:permanent:<id>` containing `{ id: 'meta:permanent:<id>', name: string, updatedAt: number }`
   - OR simpler approach: store the name as part of the note ID using `permanent:<uuid>` and maintain a separate list. Actually, simplest: use `permanent:<uuid>` as ID, query all permanent notes, store display name in a metadata key `permnames` in the notes store as `{ id: 'setting:permanentNames', value: Record<string, string> }` via usePersistedState.
   - RECOMMENDED approach: Use `permanent:<uuid>` for note IDs. Maintain a names map in `usePersistedState('setting:permanentNames', {})` as `Record<string, string>` mapping noteId -> display name. This avoids needing to read each note just to list names.
   - `createNote(name)`: generate UUID via `crypto.randomUUID()`, create note ID `permanent:${uuid}`, save empty doc content via `getDatabase()` + `db.put()`, add to names map, select the new note. Return the note ID.
   - `deleteNote(noteId)`: delete from IndexedDB via `db.delete(NOTES_STORE, noteId)`, remove from names map, if deleted note was selected, select first remaining or null.
   - `selectNote(noteId)`: update selectedNoteId state.
   - Use `usePersistedState('setting:selectedPermanentNote', null)` to persist which note is selected across sessions.
