2. Create `src/storage/database.ts`:
```typescript
import { type IDBPDatabase, openDB } from 'idb';

const DB_NAME = 'paneful-notes';
const DB_VERSION = 1;
const STORE_NAME = 'notes';

export type PanefulDB = IDBPDatabase<unknown>;

export const NOTES_STORE = STORE_NAME;

export async function getDatabase(): Promise<PanefulDB> {
  return openDB(DB_NAME, DB_VERSION, {
    upgrade(db) {
      if (!db.objectStoreNames.contains(STORE_NAME)) {
        db.createObjectStore(STORE_NAME, { keyPath: 'id' });
      }
    },
  });
}
````

Uses the `idb` library (not `idb-keyval`) per CONTEXT.md decision. The `openDB` function handles database creation and schema upgrades. The store uses `id` as keyPath.

3. Create `src/storage/use-note.ts` -- a React hook implementing the hook-only storage pattern from CONTEXT.md:

   The hook must:
   - Accept a `noteId` parameter (defaults to `'current'` for Phase 1)
   - Return `{ content: JSONContent | null; error: string | null; loading: boolean; saveContent: (content: JSONContent) => void }`
   - On mount: open database, read note by id, set content state (or null if not found)
   - `saveContent`: debounce 300ms, then write `{ id, content, updatedAt: Date.now() }` to IndexedDB
   - On blur (window `blur` event): flush pending save immediately (clear debounce timer and save now)
   - On `visibilitychange` to hidden: flush pending save immediately
   - On `beforeunload`: best-effort flush
   - On IndexedDB write failure: set error state with message "Could not save your note. Check that your browser has available storage space." (per UI-SPEC copywriting)
   - Clean up debounce timer and event listeners on unmount
   - Store pending content in a ref so flush always saves latest content

   Implementation notes:
   - Use `useRef` for debounce timer and pending content
   - Use `useCallback` for saveContent to maintain stable reference
   - Use `useEffect` for mount/unmount lifecycle (load on mount, cleanup on unmount)
   - Use `useEffect` for blur/visibilitychange/beforeunload event listeners
   - The 300ms debounce value should be a constant `SAVE_DEBOUNCE_MS = 300`
   - All functions must use named exports, no default exports
