1. **use-templates.ts** -- Create `useTemplates` hook:
   ```typescript
   import type { JSONContent } from '@tiptap/react';

   interface Template {
     content: JSONContent;
     id: string;
     name: string;
   }

   interface TemplateDefaults {
     weekday: string | null;   // template ID
     weekend: string | null;   // template ID
     weekly: string | null;    // template ID
   }

   interface UseTemplatesResult {
     defaults: TemplateDefaults;
     deleteTemplate: (templateId: string) => void;
     getTemplateContent: (templateId: string) => JSONContent | null;
     renameTemplate: (templateId: string, newName: string) => void;
     saveTemplate: (name: string, content: JSONContent) => string;
     setDefault: (slot: 'weekday' | 'weekend' | 'weekly', templateId: string | null) => void;
     templates: ReadonlyArray<Template>;
   }
   ```
   Storage approach:
   - Templates stored in IndexedDB notes store with key format `template:<uuid>` (same store, different prefix)
   - Template content stored as `{ content: JSONContent, id: 'template:<uuid>', name: string, updatedAt: number }`
   - Template list loaded via `IDBKeyRange.bound('template:', 'template:\uffff')` (same pattern as permanent notes)
   - Defaults stored via `usePersistedState('setting:templateDefaults', { weekday: null, weekend: null, weekly: null })`
   - For workspace scoping: prefix template keys with workspace when not 'personal' (same as note ID convention from use-workspace-note-id.ts). For now, templates use unprefixed keys (personal workspace).
   - `saveTemplate`: generates `template:<uuid>` ID, puts content in DB, updates local state
   - `renameTemplate`: updates the stored record's name field
   - `deleteTemplate`: removes from DB, clears from defaults if assigned
   - `getTemplateContent`: returns content for a template ID (from local state, already loaded)

2. **use-workspaces.ts** -- Create `useWorkspaces` hook:
   ```typescript
   interface Workspace {
     id: string;
     name: string;
   }

   interface UseWorkspacesResult {
     activeWorkspace: Workspace;
     createWorkspace: (name: string) => string;
     deleteWorkspace: (workspaceId: string) => void;
     renameWorkspace: (workspaceId: string, name: string) => void;
     switchWorkspace: (workspaceId: string) => void;
     workspaces: ReadonlyArray<Workspace>;
   }
   ```
   Storage approach:
   - Workspace list stored via `usePersistedState('setting:workspaces', [{ id: 'personal', name: 'Personal' }])`
   - `createWorkspace`: generates `ws-<uuid>` ID, adds to workspace list
   - `switchWorkspace`: calls `setActiveWorkspaceId` from WorkspaceContext, flushes any pending saves
   - `deleteWorkspace`: shows confirm dialog text (per UI-SPEC: "All notes, templates, and settings in this workspace will be permanently deleted. This cannot be undone."), removes workspace from list, deletes all keys with `ws:${id}:` prefix from IndexedDB
   - Cannot delete the last remaining workspace
   - `renameWorkspace`: updates name in workspace list
