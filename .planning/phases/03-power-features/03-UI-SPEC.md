# Phase 3 — UI Design Contract

> Visual and interaction contract for the power features phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none (inline SVG for search icon, back arrow, open-in-editor icon) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` |

**Rationale:** Phase 3 continues the plain CSS approach established in Phases 1 and 2. All new components (command palette, note browser, workspace label, back-to-today button) are hand-built with semantic HTML and CSS custom properties. No component library is introduced.

**Styling approach:** Plain CSS files per feature folder. CSS custom properties for all theme-able values. No Tailwind, no CSS modules. Files stay under 100 lines per DEVX-03.

## Color

Phase 3 inherits the full dual-theme color system from Phase 2. No new color tokens are introduced. All new elements use existing CSS custom properties.

### New Element Color Assignments

| Element | Light | Dark | CSS Variable |
|---------|-------|------|-------------|
| Palette backdrop | `rgba(0, 0, 0, 0.4)` | `rgba(0, 0, 0, 0.6)` | New: `--color-backdrop` |
| Palette surface | `#ffffff` | `#292524` | `--color-bg` / `--color-bg-secondary` |
| Palette input background | `#ffffff` | `#292524` | `--color-input-bg` |
| Palette input text | `#1a1a1a` | `#e5e5e5` | `--color-input-text` |
| Palette result hover | `#f5f5f5` | `#292524` | `--color-bg-secondary` |
| Palette result active/selected | `#f5f5f5` | `#292524` | `--color-bg-secondary` |
| Search highlight (match text) | `#fef3cd` | `#451a03` | `--color-warning-bg` (reused as highlight) |
| Workspace label text | `#6b7280` | `#a8a29e` | `--color-text-secondary` |
| "Back to today" text | `#2563eb` | `#60a5fa` | `--color-focus` (reused as link color) |
| Date reference link text | `#2563eb` | `#60a5fa` | `--color-focus` (reused as link color) |
| Note browser item text | `#1a1a1a` | `#e5e5e5` | `--color-text` |
| Note browser item secondary | `#6b7280` | `#a8a29e` | `--color-text-secondary` |

### New CSS Custom Properties

| Property | Light | Dark |
|----------|-------|------|
| `--color-backdrop` | `rgba(0, 0, 0, 0.4)` | `rgba(0, 0, 0, 0.6)` |
| `--color-highlight` | `#fef3cd` | `#451a03` |
| `--color-link` | `#2563eb` | `#60a5fa` |

**Rationale for `--color-link`:** Reuses the focus ring color values for interactive links (date references, "Back to today"). This maintains the existing monochrome aesthetic where the only color departure from the neutral palette is the blue focus/link color.

Accent reserved for: today circle on calendar, selected day fill on calendar (unchanged from Phase 2). Never used for links, palette highlights, or general interactive elements.

## Interaction Contract

### Command Palette

| Interaction | Behavior |
|-------------|----------|
| CMD+K | Opens palette. Focus moves to input field. |
| Escape | Closes palette. Focus returns to previously active element. |
| Click backdrop | Closes palette. Focus returns to previously active element. |
| Type text | Debounced search (200ms). Results update below input. |
| Type `>` as first character | Switches to command mode. Shows filtered command list. |
| Arrow Up/Down | Navigate between results. Active result highlighted with `--color-bg-secondary`. |
| Enter on result | Executes result action (navigate to note or run command). Closes palette. |
| Enter on search result | Opens note in left pane main editor. Closes palette. |

### Note Browser

| Interaction | Behavior |
|-------------|----------|
| Click note item | Selects note for preview in a mini inline editor within the section (same as current permanent note behavior, but with list instead of dropdown). |
| Click "Open in editor" icon | Opens note in left pane main editor. Left pane shows "Back to today" button. |
| Hover note item | Shows `--color-bg-secondary` background and reveals "Open in editor" icon. |

### "Back to Today" Navigation

| Interaction | Behavior |
|-------------|----------|
| Click "Back to today" | Left pane navigates back to today's daily note. Button disappears. |
| Any note navigation | Left pane shows the navigated note. "Back to today" appears if note is not today's daily note. |

### Date Reference Links

| Interaction | Behavior |
|-------------|----------|
| Type `[[YYYY-MM-DD]]` | Input rule transforms text into a clickable date reference node. |
| Click date reference | Navigates to that day's daily note in the left pane. "Back to today" appears. |
| Invalid date typed | `[[invalid]]` remains as plain text -- no transformation occurs. |

### Image Paste

| Interaction | Behavior |
|-------------|----------|
| CMD+V with image in clipboard | Image is stored in IndexedDB blob store. Image node is inserted at cursor position in editor. |
| Image loading | Display immediately using `URL.createObjectURL()` from the stored blob. |
| Image in exported markdown | Blob URL rewritten to relative path `../images/<filename>.png`. |

### Export/Import

| Interaction | Behavior |
|-------------|----------|
| "Export Data" command | Generates zip file. Browser download dialog appears. Live region announces completion. |
| "Import Data" command | Opens native file picker (accepts `.zip` and `.md`). Processing happens synchronously for small files. |
| Import conflict (note exists) | Appended below existing content with `## Imported on {date}` heading. No confirmation dialog -- non-destructive by design. |
| Import completion | Live region announces "{N} notes imported". Palette closes. |

### Keyboard Navigation (additions to Phase 2)

| Shortcut | Action |
|----------|--------|
| `CMD+K` | Open command palette |
| `Escape` | Close command palette (when open) |
| `Arrow Up/Down` | Navigate palette results (when palette open) |
| `Enter` | Select palette result (when palette open) |

### Focus Management (additions to Phase 2)

| Scenario | Focus Behavior |
|----------|---------------|
| CMD+K pressed | Save `document.activeElement`. Focus palette input. |
| Palette closed (Escape/backdrop) | Restore focus to saved element. |
| Palette result selected | Close palette. Focus target element (editor for note navigation). |
| "Back to today" clicked | Focus daily note editor. |
| Workspace switched | Focus daily note editor with new workspace content. |
| Template saved/deleted | Focus remains in palette (if palette is open). |

### Accessibility (additions to Phase 2)

| Element | ARIA |
|---------|------|
| Palette container | `role="dialog"`, `aria-modal="true"`, `aria-label="Command palette"` |
| Palette input | `role="combobox"`, `aria-expanded="true"`, `aria-controls="palette-results"`, `aria-activedescendant` tracks active result |
| Palette results list | `role="listbox"`, `id="palette-results"` |
| Palette result item | `role="option"`, `aria-selected` for active item |
| "Back to today" button | `<a>` or `<button>` with text content (self-labeling) |
| Workspace label | `aria-label="Current workspace"` |
| Note browser list | `role="listbox"`, `aria-label="Permanent notes"` |
| Note browser item | `role="option"` |
| Date reference link | `<a>` with `aria-label="Go to {formatted date}"` |
| Inline image | `<img>` with `alt=""` (decorative default), user-editable alt text |
